# おがわやチェックリスト 非IT担当者向け運用説明書

この説明書は、プログラムを書かない運用担当者が、おがわやチェックリストを日常運用するための資料です。
現在の運用は、GASやGoogleスプレッドシートを正本にせず、GitHub Pages、Firebase Auth、Firestore、GitHub Actionsで動きます。

## 1. 開く場所

| 用途 | URL |
| --- | --- |
| 従業員がLINEで開く画面 | `https://liff.line.me/2009859108-sJ31BCFx` |
| 従業員用Web画面 | `https://tapioka0112.github.io/ogawaya/` |
| 管理者画面 | `https://tapioka0112.github.io/ogawaya/admin.html` |
| Firebase Console | `https://console.firebase.google.com/project/owagaya-fd93b/overview` |
| GitHub Actions | `https://github.com/tapioka0112/ogawaya/actions` |

普段使うのは、従業員用画面と管理者画面です。Firebase Console と GitHub Actions は、管理者追加や障害確認のときだけ使います。

## 2. システムの仕様

| 項目 | 内容 |
| --- | --- |
| 店舗ID | `store-hashimoto` |
| データの正本 | Firestore |
| 画面の公開場所 | GitHub Pages |
| 従業員の本人確認 | LINE LIFF と Firebase Auth |
| 管理者ログイン | Firebase Auth のメールアドレスとパスワード |
| 管理者権限 | Firestore の `stores/store-hashimoto/admins/{uid}` にUIDが登録されている人 |
| 日次タスク作成 | GitHub Actions の `Daily start` |
| 未完了通知 | GitHub Actions の `Incomplete reminder` |

## 3. 日付とタスク作成の考え方

このアプリでは、日本時間 `10:30` を営業日の切り替え時刻として扱います。

| 処理 | タイミング | 内容 |
| --- | --- | --- |
| 日次タスク作成 | 毎日 `10:35` | その日のタスクをFirestoreに作成します。 |
| 未完了通知 | 毎日 `00:35` | 前日分の未完了タスクをLINEへ通知します。 |

タスク種別ごとの自動作成ルールです。

| 種別 | 自動作成される日 |
| --- | --- |
| 日間 | 毎日 |
| 週間 | 日曜日 |
| 月間 | 毎月1日 |

管理者が10:30より前に当日分のタスクを手動で入れた場合でも、従業員画面は当日分にタスクがあれば当日分を表示します。

## 4. 標準テンプレート

現在の標準テンプレートは次の3つです。

| テンプレート | 件数 | 主な用途 |
| --- | ---: | --- |
| 日間清掃チェックリスト | 6件 | 毎日の清掃確認 |
| 週間清掃チェックリスト | 10件 | 週ごとの清掃確認 |
| 月間清掃チェックリスト | 4件 | 月ごとの清掃確認 |

テンプレートは、管理者画面の「テンプレートを挿入」から指定日に追加します。

## 5. 従業員の日常操作

1. LINEから `https://liff.line.me/2009859108-sJ31BCFx` を開きます。
2. 表示されたタスクを確認します。
3. 終わったタスクを押してチェック済みにします。
4. 間違えて押した場合は、もう一度押して未完了に戻します。
5. 日間、週間、月間のタブを切り替えて、それぞれのタスクを確認します。
6. 統計タブで、自分の完了数と月間の達成状況を確認します。

チェック操作はFirestoreへ直接保存されます。通信状態が悪いときは反映に時間がかかるため、画面を閉じずに数秒待ってください。

## 6. 管理者の日常操作

管理者画面は `https://tapioka0112.github.io/ogawaya/admin.html` です。

### ログイン

1. 管理者画面を開きます。
2. Firebase Auth に登録された管理者メールアドレスを入力します。
3. パスワードを入力します。
4. ログインします。

ログインできる条件は2つあります。

| 条件 | 内容 |
| --- | --- |
| Firebase Auth ユーザーがある | メールアドレスとパスワードでユーザーが作成されていること |
| 管理者allowlistにUIDがある | `stores/store-hashimoto/admins/{uid}` がFirestoreに存在すること |

### タスクを作成

「タスクを作成」は、後で日付に入れるためのタスク部品を作る操作です。作成しただけでは、その日のチェックリストには入りません。

1. 「タスクを作成」を押します。
2. タスク名を入力します。
3. 必要なら詳細を入力します。
4. 種別を日間、週間、月間から選びます。
5. 「タスクを作成」を押します。

### タスクを挿入

「タスクを挿入」は、作成済みのタスク部品を指定日に1件追加する操作です。

1. 対象日を選びます。
2. 「タスクを挿入」を押します。
3. 追加するタスクを選びます。
4. 「選択したタスクを挿入」を押します。
5. タスク一覧に表示されたことを確認します。

### テンプレートを作成

テンプレートは、複数タスクをまとめて追加するためのセットです。

1. 先に必要なタスクを作成します。
2. 「テンプレートを作成」を押します。
3. テンプレート名を入力します。
4. 種別を選びます。
5. 含めたいタスクにチェックを付けます。
6. 「テンプレートを作成」を押します。

### テンプレートを挿入

1. 対象日を選びます。
2. 「テンプレートを挿入」を押します。
3. 挿入するテンプレートを選びます。
4. 「選択したテンプレートを挿入」を押します。
5. タスク一覧に追加されたことを確認します。

テンプレート挿入はFirestoreへ直接保存されます。管理者画面には先に反映され、従業員画面にも同期されます。

### タスクを削除

削除は、その日のチェックリストから外す操作です。タスク部品やテンプレート自体は削除しません。

1. 対象日を選びます。
2. 消したいタスクの「削除」を押します。
3. 一覧から消えたことを確認します。

削除は管理者画面では即時反映されます。他の端末にはFirestoreの同期で反映されます。

## 7. 管理者を追加する方法

1. Firebase Console を開きます。
2. `Authentication` を開きます。
3. `Users` で管理者用ユーザーを追加します。
4. 作成されたユーザーのUIDをコピーします。
5. `Firestore Database` を開きます。
6. `stores` → `store-hashimoto` を開きます。
7. `admins` コレクションを作ります。既にある場合は開きます。
8. ドキュメントIDに、コピーしたUIDを入れてドキュメントを作ります。

推奨フィールドです。

| フィールド | 値の例 |
| --- | --- |
| `email` | 管理者メールアドレス |
| `role` | `admin` |
| `createdAt` | 登録日 |

重要なのは、`admins` のドキュメントIDが Firebase Auth のUIDと完全一致していることです。

## 8. 自動処理の確認方法

GitHub Actions の `https://github.com/tapioka0112/ogawaya/actions` を開きます。

| Workflow | 役割 | 正常な状態 |
| --- | --- | --- |
| `Deploy LIFF Pages` | 画面をGitHub Pagesへ公開 | `success` |
| `Daily start` | 毎日10:35にタスクを作成 | `success` |
| `Incomplete reminder` | 毎日00:35に未完了通知 | `success` |

`failure` が出ている場合は、担当エンジニアへ連絡してください。非IT担当者は失敗ログの削除や再設定をしないでください。

## 9. よくあるトラブル

### 管理者画面に入れない

確認すること:

1. メールアドレスとパスワードが正しいか。
2. Firebase Auth にそのユーザーが存在するか。
3. Firestore の `stores/store-hashimoto/admins/{uid}` が存在するか。

### テンプレート一覧が空

確認すること:

1. 管理者画面をリロードします。
2. Firestore の `stores/store-hashimoto/templates` に active なテンプレートがあるか確認します。
3. 標準テンプレートがない場合は、担当エンジニアへ復旧を依頼します。

### ホーム画面にタスクが出ない

確認すること:

1. 画面をリロードします。
2. 管理者画面で対象日にタスクが入っているか確認します。
3. GitHub Actions の `Daily start` が成功しているか確認します。

10:30前に当日分を手動挿入した場合、当日分にタスクが存在すれば従業員画面に表示されます。

### チェックや削除の反映が遅い

確認すること:

1. 通信状態を確認します。
2. 数秒待ってから画面をリロードします。
3. 別端末でも同じ状態か確認します。

チェック、テンプレート挿入、削除はFirestoreへ直接書き込みます。GAS待ちではありません。

### 未完了通知が来ない

確認すること:

1. GitHub Actions の `Incomplete reminder` が成功しているか確認します。
2. LINEの通知先ユーザーが一度LIFF画面を開いているか確認します。
3. `LINE_CHANNEL_ACCESS_TOKEN` の設定は非IT担当者が触らず、担当エンジニアへ確認を依頼します。

## 10. 触ってよいもの・触ってはいけないもの

触ってよいものです。

| 場所 | 操作 |
| --- | --- |
| 従業員画面 | タスクのチェック、統計確認 |
| 管理者画面 | タスク作成、タスク挿入、テンプレート作成、テンプレート挿入、日付内タスク削除 |
| Firebase Authentication | 管理者ユーザーの追加 |
| Firestore `admins` | 管理者UIDの追加 |

触ってはいけないものです。

| 場所 | 理由 |
| --- | --- |
| Firestore rules | 権限設定が壊れると全員が使えなくなります。 |
| GitHub Actions secrets | 通知やFirestore書き込みが止まります。 |
| `stores/store-hashimoto/runs` の直接編集 | チェック状態や統計が壊れます。 |
| `stores/store-hashimoto/users` の直接編集 | LINE通知先が壊れます。 |
| GitHub のコード | 画面や自動処理の動作が変わります。 |

## 11. 担当エンジニアへ伝えるときの情報

障害連絡では、次の情報を送ってください。

| 項目 | 例 |
| --- | --- |
| 発生日時 | `2026-05-11 10:40` |
| 画面 | 従業員画面、管理者画面、Firebase Consoleなど |
| 操作 | テンプレート挿入、削除、チェックなど |
| 対象日 | `2026-05-11` |
| エラー文 | 画面に出た文言をそのまま送る |
| スクリーンショット | 可能なら添付 |

「何を押したら、何が起きたか」を短く正確に伝えると復旧が早くなります。
